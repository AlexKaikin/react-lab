# Стек

# Архитектура (FSD)

Проект построен по методологии **Feature-Sliced Design (FSD)**. Код организован в слои, каждый из которых решает свою задачу и знает только о нижележащих слоях.

## Слои (Layers)

| Слой       | Назначение                                     | Примеры из проекта             |
| ---------- | ---------------------------------------------- | ------------------------------ |
| `app`      | Next.js App Router: страницы, лэйауты, роутинг | `layout.tsx`, `page.tsx`       |
| `pages`    | Страницы приложения, композиция виджетов       | `home`                         |
| `widgets`  | Композиционные блоки из фич и сущностей        | `header`                       |
| `features` | Пользовательские сценарии (взаимодействия)     | `theme`                        |
| `entities` | Бизнес-сущности                                | _пока отсутствует_             |
| `shared`   | Переиспользуемый код: UI-кит, утилиты, типы    | `button`, `icon`, `classNames` |

## Правила зависимостей

Слои импортируют код только из слоёв **ниже** по иерархии. Обратные и горизонтальные импорты между слоями одного уровня запрещены.

## Сегменты (Segments)

Каждый слайс внутри слоя делится на сегменты:

| Сегмент  | Назначение                        | Примеры                                    |
| -------- | --------------------------------- | ------------------------------------------ |
| `ui`     | React-компоненты и отрисовка      | `header.tsx`, `theme-provider.tsx`         |
| `model`  | Состояние, сторы, типы, константы | `use-theme.ts`, `types.ts`, `constants.ts` |
| `lib`    | Вспомогательные функции и утилиты | `classNames.ts`                            |
| `api`    | Запросы к API                     | _пока отсутствует_                         |
| `config` | Конфигурация слайса               | _пока отсутствует_                         |

## Публичное API (index.ts)

Каждый слайс экспортирует содержимое через `index.ts`. Внутренняя реализация скрыта.

# Принципы написания кода

## Вёрстка

- Семантические теги
- Доступность (a11y)
- Валидность W3C

## Скрипты

- Строгая типизация (TypeScript)
- Производительность: мемоизация, динамический импорт, Server/Client разделение
- Безопасность: XSS, CSRF, валидация на границах
- Принципы: DRY / SOLID / KISS
- Состояние: приоритет Server State, минимум Client State
- Оптимизация рендера: изображения, шрифты, CLS
- **Ленивая загрузка модалок**: компоненты, передаваемые в `openModal`, должны импортироваться через `next/dynamic` или `React.lazy`, чтобы не увеличивать бандл страницы. Исключение — если модалка гарантированно весит <1KB и будет вызвана сразу.
